# NeedMarket — Style Spec для Claude Code

Спецификация нового визуального стиля мини-аппа. Цель: переодеть существующие экраны (см. `UI_INVENTORY.md`) в новый стиль и добавить нижнюю таб-навигацию, **не меняя бизнес-логику**.

**Источник истины по визуалу** — три HTML-макета (открой и сверяйся по точным значениям CSS):
- `needmarket-home.html` — лендинг компании (бренд-строка, hero, «как это работает», карточки, таб-бар + FAB)
- `needmarket-shells.html` — три шелла рядом (блогер / компания / админ) с таб-барами
- `needmarket-create-lot.html` — форма создания лота (поля, чипы, степпер, дата, загрузка, MainButton)

Стек, под который реализуем: React 18 + Vite + `@tma.js/sdk-react` + `@telegram-apps/telegram-ui`, тема через `themeParams.isDark` + `bindCssVars`. Детали значений автор подгонит на MVP — здесь зафиксированы направление и токены.

---

## 1. Принципы

- Чистый, скруглённый, «потребительский финтех». Белые карточки на светло-сером фоне, мягкие тени, много воздуха.
- Один акцент — бренд-синий. Им красятся: главное действие, активный таб, ссылки, FAB, hero.
- Статусы — цветные пилюли (плашки). Цвет несёт смысл (в работе / ждёт оплаты / активен / завершён).
- Иконки — единый line-набор (Lucide), **никаких эмодзи**.
- Жирные крупные заголовки (вес 800), спокойный тело-текст. Sentence case везде, без CAPS.
- Тёмная тема обязательна: бренд-синий и FG-цвета статусов почти не меняются, меняются фоны/текст/границы и тинты плашек.

## 2. Токены (CSS variables)

Заводим отдельный слой `--nm-*` поверх telegram-ui. Бренд и статусы — фиксированные; поверхности/текст/границы имеют значения для light и dark.

```css
:root{
  /* brand */
  --nm-blue:#2F7CF6;
  --nm-blue-strong:#2563EB;     /* конец градиента, итоговые суммы */
  --nm-blue-soft:#E9F1FF;       /* мягкие заливки: тайлы, soft-пилюли */
  --nm-blue-line:#CFE0FB;       /* бордер ghost-кнопки */
  --nm-grad:linear-gradient(135deg,#4E93FF 0%,#2563EB 100%); /* hero, FAB, логотип */

  /* surfaces & text (light) */
  --nm-bg:#F2F3F6;              /* фон страницы/секции */
  --nm-surface:#FFFFFF;         /* карточки, шапки, таб-бар */
  --nm-surface-2:#F2F3F6;       /* инсет-блоки: разбивка, строка блогера */
  --nm-ink:#14161B;             /* основной текст */
  --nm-ink-2:#8A909B;           /* вторичный текст */
  --nm-ink-3:#AAB0BA;           /* третичный / placeholder */
  --nm-line:#ECEEF2;            /* границы, разделители */

  /* status */
  --nm-green:#1FA971;  --nm-green-bg:#E4F7EE;   /* «В работе» / успех */
  --nm-amber:#DD8A0B;  --nm-amber-bg:#FBEFD6;   /* «Ждёт оплаты» */
  --nm-info:#2F7CF6;   --nm-info-bg:#E6F0FF;    /* «Активен» / «Модерация» / «К выплате» */
  --nm-neutral:#7D838F; --nm-neutral-bg:#EEF0F3;/* «Завершён» */
  --nm-red:#FF4D67;                              /* бейджи, «лайк»-чип */

  /* user avatar gradient (фиолетовый) */
  --nm-ava-a:#8E7BF5;  --nm-ava-b:#6A55E6;

  /* radii */
  --nm-r-card:18px;
  --nm-r-field:13px;     /* инпуты, кнопки */
  --nm-r-tile:17px;      /* иконка-тайл «как это работает» */
  --nm-r-pill:20px;      /* фильтр-чипы */
  --nm-r-badge:9px;      /* статус-пилюли */

  /* elevation */
  --nm-sh-card:0 2px 10px rgba(22,30,55,.04);
  --nm-sh-hero:0 14px 30px rgba(37,99,235,.32);
  --nm-sh-fab:0 9px 20px rgba(37,99,235,.42);
  --nm-sh-btn:0 6px 14px rgba(47,124,246,.26);
}

/* dark: вешать на тот же сигнал, что и telegram-ui (themeParams.isDark) */
:root[data-theme="dark"]{
  --nm-blue:#3B82F6;
  --nm-blue-strong:#2F7CF6;
  --nm-blue-soft:#16233A;
  --nm-blue-line:#2C3F5E;

  --nm-bg:#0F1115;
  --nm-surface:#1C1E24;
  --nm-surface-2:#24262E;
  --nm-ink:#F1F3F6;
  --nm-ink-2:#9AA1AD;
  --nm-ink-3:#6A7079;
  --nm-line:#2A2D35;

  --nm-green:#34C98A;  --nm-green-bg:#13302625;
  --nm-amber:#F0A93C;  --nm-amber-bg:#33260F;
  --nm-info:#5C9BFF;   --nm-info-bg:#16233A;
  --nm-neutral:#9AA1AD; --nm-neutral-bg:#262931;
  --nm-red:#FF6178;
}
```

Замечания по интеграции:
- Тени в тёмной теме почти не видны — это нормально; разделение даёт `--nm-line` и `--nm-surface` vs `--nm-bg`. Можно слегка усилить границы карточек в dark.
- Если хочется идеально совпасть с темой Telegram, `--nm-bg` / `--nm-surface` / `--nm-ink*` можно зашить на `--tgui--*` переменные вместо хардкода. Бренд (`--nm-blue*`) и статусы оставить фиксированными.
- Сигнал темы: проще всего ставить `data-theme="dark"` на `<html>` из того же места, где уже читается `themeParams.isDark`.

## 3. Типографика

Системный стек (на мобиле это SF Pro / Roboto): `-apple-system, BlinkMacSystemFont, "SF Pro Display", "Segoe UI", Roboto, sans-serif`.

| Роль | Размер | Вес |
|---|---|---|
| Заголовок экрана / секции | 21 / 19 | 800 |
| Заголовок карточки | 15.5–16 | 700 |
| Тело | 13–15 | 400–600 |
| Подпись / мета | 11–12.5 | 500–600 |
| Лейбл поля | 12.5 | 600 |

Буквенный трекинг у крупных заголовков чуть отрицательный (`letter-spacing:-.3px`). telegram-ui-компоненты, которые мы переодеваем, получают наш тип через классы-обёртки.

## 4. Иконки

Поставить `lucide-react` (line-набор, один цвет, наследует `color`/размер). Заменить все эмодзи. Размеры: 24px в таб-баре, 22px в шапке, 25–27px в тайлах, 14px внутри чипа/пилюли, 18–20px инлайн.

| Контекст | Иконка (Lucide) |
|---|---|
| Уведомления (шапка) | `Bell` (+ синяя точка непрочитанного) |
| Назад (вложенный экран) | нативный Telegram BackButton |
| Чип выбран | `Check` |
| Карточка → деталь | `ChevronRight` |
| Дата | `Calendar` · раскрытие `ChevronDown` |
| Копировать контакт | `Copy` |
| Загрузка ТЗ | `UploadCloud` (или `Upload`) |
| Hero | `Megaphone` |
| Чипы метрик в hero | `Heart` / `Eye` / `Users` |
| «Как это работает» 1–5 | `FileText` / `ShieldCheck` / `Users` / `MessageSquare` / `BarChart3` |
| Персональный менеджер | `Headset` |

**Иконки таб-баров** — см. таблицы навигации в §6.

## 5. Компоненты (рецепты)

Ключевые примитивы ниже. Остальное (hero, тайлы «как это работает», стопка аватаров, карточка менеджера) — забирай точные значения из `needmarket-home.html`; карточки лотов и админа — из `needmarket-shells.html`; контролы формы — из `needmarket-create-lot.html`.

### 5.1 Шапка экрана (root)
Слева — заголовок (800) + подзаголовок (12.5, `--nm-ink-2`). Справа — колокол (квадрат 42px, `--nm-r-field`, бордер, синяя точка) + аватар пользователя (40px круг, градиент `--nm-ava-a/b`). Фон `--nm-surface`.

### 5.2 Статус-пилюля
```css
.nm-pill{display:inline-flex;align-items:center;font-size:11.5px;font-weight:700;
  padding:4px 10px;border-radius:var(--nm-r-badge);}
.nm-pill.green  {color:var(--nm-green);  background:var(--nm-green-bg);}
.nm-pill.amber  {color:var(--nm-amber);  background:var(--nm-amber-bg);}
.nm-pill.info   {color:var(--nm-info);   background:var(--nm-info-bg);}
.nm-pill.neutral{color:var(--nm-neutral);background:var(--nm-neutral-bg);}
```
Маппинг статусов лота: `active`→info «Активен», `in_progress`→green «В работе», `awaiting_payment`→amber «Ждёт оплаты», `awaiting_payout`→info «К выплате», `completed`→neutral «Завершён», `disputed`→red.

### 5.3 SelectChip (фильтр / мультивыбор)
Базовый — белый с бордером; выбранный — синяя заливка, белый текст, иконка `Check`. Может нести счётчик.
```css
.nm-chip{font-size:13px;font-weight:600;padding:9px 14px;border-radius:var(--nm-r-pill);
  background:var(--nm-surface);border:1px solid var(--nm-line);color:var(--nm-ink);
  display:inline-flex;align-items:center;gap:6px;white-space:nowrap;}
.nm-chip[data-on="true"]{background:var(--nm-blue);border-color:var(--nm-blue);color:#fff;}
.nm-chip .count{color:var(--nm-ink-3);}            /* в on-состоянии: rgba(255,255,255,.7) */
```
Ряд фильтров скроллится по горизонтали; в форме чипы переносятся (`flex-wrap`).

### 5.4 Карточка лота (список/лента)
Белая, `--nm-r-card`, бордер `--nm-line`, тень `--nm-sh-card`, паддинг 15–16.
Шапка: аватар-тайл 42px (квадрат `--nm-r-tile` у компании/админа, круг у блогера) + колонка [заголовок 15.5/700 · подзаголовок 12.5/`--nm-ink-2`] + статус-пилюля справа.
Низ (через разделитель `--nm-line`): слева блок «Бюджет» (лейбл 11/`ink-2` + сумма 16.5/800), справа мета (`Откликов: N · выбрано M/K` и/или дедлайн `до 25 июня`).
- **Блогер** (открытые лоты): подзаголовок = «Компания · Категория · Площадки», справа soft-пилюля `«3 места»`, низ = бюджет + дедлайн.
- **Компания** (свои лоты): заголовок = название лота, статус-пилюля по статусу, низ = бюджет + `Откликов … · выбрано …` + дедлайн.

### 5.5 Карточки админа
- **Выплата**: шапка + блок разбивки (§5.6) + строка блогера (`--nm-surface-2`, аватар + имя + «Категория · @ник» + ghost-кнопка «Связаться») + filled-кнопка во всю ширину «Закрыть лот».
- **Оплата**: шапка + строки key/value (`Компания`, `Бюджет`, `Контакт` с `@ник` + иконкой `Copy`) + ряд из двух кнопок [ghost «Связаться»][filled «Активировать лот»].

### 5.6 Блок разбивки (деньги)
Инсет `--nm-surface-2`, `--nm-r-field`, паддинг 11–13. Строки `space-between` (13px): лейбл `--nm-ink-2`, значение 600. Итоговая строка — верхняя граница `--nm-line`, лейбл `--nm-ink`/700, значение `--nm-blue-strong`/800/15.
Пример: `Бюджет компании 200 000 ₸` · `Комиссия 10% −20 000 ₸` · `К выплате блогеру 180 000 ₸`.

### 5.7 Кнопки
```css
.nm-btn{display:inline-flex;align-items:center;justify-content:center;gap:6px;
  font-weight:700;font-size:14px;padding:11px 16px;border-radius:var(--nm-r-field);border:1px solid transparent;}
.nm-btn.fill {background:var(--nm-blue);color:#fff;box-shadow:var(--nm-sh-btn);}
.nm-btn.ghost{background:var(--nm-surface);color:var(--nm-blue);border-color:var(--nm-blue-line);}
.nm-btn.sm   {padding:8px 13px;font-size:12.5px;border-radius:10px;}
```
**Главное действие экрана — нативная Telegram MainButton**, а не наша кнопка (см. §6). Наши `.fill/.ghost` — для вторичных действий внутри карточек (выбрать/отклонить/связаться/активировать и т.п.).

### 5.8 Контролы формы
- Инпут / textarea: `--nm-surface`, бордер `--nm-line`, `--nm-r-field`, паддинг 13×14, текст 15, placeholder `--nm-ink-3`.
- Поле с суффиксом (бюджет ₸) и поле-дата (иконка `Calendar` слева, значение, `ChevronDown` справа) — строка `.inp-row` с теми же рамкой/радиусом.
- Степпер «сколько блогеров»: бордер-контейнер, кнопки −/+ (синяя иконка) по бокам значения (16/700).
- Зона загрузки ТЗ: пунктирный бордер `--nm-blue-line`, фон `--nm-blue-soft`, по центру иконка-тайл + «Прикрепить файлы» + подпись.
Лейблы полей 12.5/600/`--nm-ink-2`; необязательность — `em` серым. Секции формы: мини-заголовок 13/800 с синей точкой-маркером.

### 5.9 Лендинг (опционально, если оставляем)
Бренд-строка (логотип NM 48px градиент + название/слоган + колокол + аватар), hero-карточка (`--nm-grad`, заголовок 24/800 белый, белая CTA-пилюля, иллюстрация = `Megaphone` + чипы метрик), ряд «как это работает» (5 тайлов с синими номерами), карточка менеджера. Всё — в `needmarket-home.html`.

## 6. Навигация — нижний таб-бар + шеллы

### Компонент `BottomTabBar`
Фон `--nm-surface`, верхняя граница `--nm-line`, паддинг ~9px сверху / 11px снизу (учесть safe-area внизу). Таб = иконка 24px + лейбл 10.5–11px/600. Активный = `--nm-blue`; неактивный = `--nm-ink-3`. Поддержать: бейдж-счётчик (красный кружок, для «Споры»), точку непрочитанного (синяя, для «Поддержка»).

**FAB (только у компании)** — приподнятая круглая кнопка 52px, фон `--nm-grad`, бордер 4px цветом `--nm-surface`, тень `--nm-sh-fab`, икнока `Plus`, смещение вверх ~−22px. Под ней лейбл «Создать».

### Правило показа таб-бара
Таб-бар виден **только на корневых экранах** табов. Любой вложенный экран (LotDetail, CreateLotForm, профиль-форма, тред поддержки) **скрывает таб-бар** и использует:
- нативную **MainButton** снизу — главное действие (Откликнуться / Опубликовать лот / Выбрать / Закрыть лот …);
- нативный **BackButton** сверху — назад.

### 6.1 Блогер — 5 табов, без FAB
Тут всё прямолинейно: каждый таб = бывшая отдельная кнопка из шапки. **Настройки уезжают в Профиль.**

| Таб | Иконка | Экран (из инвентаря) |
|---|---|---|
| Лента | `Home` | BloggerHome (лента открытых лотов) |
| Отклики | `Send` | MyResponses |
| Поиски | `Bookmark` | SavedSearches |
| Поддержка | `LifeBuoy` | SupportList → SupportThread |
| Профиль | `User` | ProfileView **+ Settings** (тумблеры уведомлений внутри) |

### 6.2 Компания — 4 таба + FAB, **настройки в Профиль**
Главное изменение IA: бывшие **«Заявки» и «Сделки» сливаются в одну «Ленту»** — компания видит только свои лоты и проваливается в свои лоты прямо из этой ленты. Отдельной вкладки «Сделки» нет; статусы лотов разруливаются фильтр-чипами внутри ленты.

| Таб | Иконка | Экран |
|---|---|---|
| Заявки | `ClipboardList` | CompanyHome — единая лента **своих** лотов (фильтр-чипы по статусу: Все / Активен / В работе / Ждёт оплаты / Завершён). Тап по карточке → LotDetail (взгляд компании) |
| Создать | `Plus` (FAB) | CreateLotForm (вложенный экран → MainButton «Опубликовать лот») |
| Поддержка | `LifeBuoy` | SupportList → SupportThread |
| Профиль | `User` | ProfileView **+ Settings** |

Раскладка бара: 4 слота, «Создать» — приподнятый FAB. Точное место FAB (по центру или вторым слотом) автор подстроит на MVP; визуально это акцентная центральная кнопка.

### 6.3 Админ — 4 таба, без FAB
Главное изменение IA: раньше **Оплаты / Выплаты / Споры жили в одном окне** (AdminPanel секциями). Теперь **каждая — отдельный экран-вкладка**.

| Таб | Иконка | Экран |
|---|---|---|
| Оплаты | `CreditCard` | Список лотов «ожидают оплаты» (AwaitingPaymentCard) → действия «Связаться / Активировать лот» |
| Выплаты | `Wallet` | Список «ожидают выплаты» (AwaitingPayoutCard) — с разбивкой комиссии 10% → «Закрыть лот» |
| Споры | `Flag` | Список споров (DisputeCard) — с разбивкой и кнопками разрешения. **Красный бейдж-счётчик** на табе |
| Поддержка | `LifeBuoy` | AdminSupportPanel (users → tickets → thread). **Синяя точка** непрочитанного |

## 7. Порядок реализации (рекомендация)

1. Завести слой токенов `--nm-*` (§2) + переключатель темы `data-theme` от `themeParams.isDark`. Подключить `lucide-react`.
2. Сделать примитивы: `StatusPill`, `SelectChip` (переодеть существующий), `LotCard`, `Button`, поля формы, `BreakdownBox`.
3. Сделать `BottomTabBar` (+ FAB, бейджи) и развести роутинг по табам трёх шеллов (§6). Завести правило «вложенный экран ⇒ скрыть таб-бар, показать MainButton».
4. Перестроить IA: компания — слить Заявки/Сделки в одну ленту; админ — разнести Оплаты/Выплаты/Споры по отдельным экранам; настройки — внутрь Профиля (блогер и компания).
5. Прогнать экраны из инвентаря через новые примитивы, сверяясь с тремя HTML-макетами.

Бизнес-логику, статусы и эндпойнты не трогаем — меняется только представление и навигация.
